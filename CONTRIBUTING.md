# Contributing

Read `AGENTS.md` and `.phoxia/project.yaml` before changing the project. Open an issue before a large or cross-domain change.

Keep changes focused, add tests for changed behavior and run `cargo test`. Update the changelog when a public version changes. Update documentation and a Feature Record when user-facing behavior or a public contract changes. Architecture decisions belong in an ADR; cross-domain contracts require RFC analysis.

Never commit credentials, personal data, private work or employer material. New dependencies must be license compatible.
